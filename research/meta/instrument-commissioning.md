## The design (K3's lean chassis + all 8 of its amendments; standing-registry-file replaced by a COMPUTED view)

**What is an instrument (A2 boundary — silent-failure-propagation test):** an artifact is an instrument iff (1) it has a deterministic/semi-deterministic read function world-state→output, (2) its output is consumed by a named decision/gate/file without full human re-derivation, (3) if it silently returned wrong output for 30 days, something downstream would break unnoticed. Pre-registered predictions are NOT instruments (their failure is loud — they get graded). Bias prose is not; its enforcement hook is (type=enforcement, certificate = code known-answer test, already covered by the L34 verifier discipline). Types: {measurement / threshold-trigger / enforcement / quota}.

**Material (A8, traceability = materiality):** an instrument is material iff its output is consumed by a named decision package, program gate, tier assignment, or notification rule. Everything else = uncertified tell, no block required.

**INSTRUMENT BLOCK (mandatory in the instrument's own canonical file — NOT a central registry; same-commit metabolism is why):**
- type · read function (one line: inputs→output) · feeds (named consumers)
- ASSUMPTION ATOMS: {term, definition, falsifying-observable} triples (A1) — the observable is the deliverable; collision checks and re-certs run on observables, not stories
- shared-inputs (denominators/datasets reused by other instruments — concentration visibility, A1) · convention/provenance (vendor, comparison convention — the BR-1 lesson)
- BIRTH HYPOTHESIS (A3 — "hypothesis," never "certificate"; a stamp is not armor): author's false-fire world(s) + false-silence world(s), each WITH observable; OR a hindsight certificate (90-day backtest) where history suffices; known-answer test where computable
- BLIND-ADVERSARY DIFF (A3): adversary agent receives ONLY the I/O spec, never author worlds; non-overlap between its list and the author's = the measured blind-spot list, appended here
- KILL CRITERION (A3): the reading pattern that retires the instrument — birth without a death-condition is how registries bloat
- CATCH LOG (A7): every blind-spot catch from ANY channel amends this block same-commit, channel-tagged {sweep|luck|principal|referee}
- escort status · last/next re-cert · anomaly state

**Operating rules:**
1. **Escort:** first 3 clean readings or first live firing (whichever later) are hand-decomposed, never auto-acted-on (K3 ADOPT verbatim).
2. **Collision check:** on STRUCTURAL ingests only (~1-3/wk) — match the datum against blocks' falsifying-observable vocabulary. Precision metric = amendments/alerts, quarterly; <1 amendment/row/quarter → loosen sensitivity deterministically (A5).
3. **Re-cert:** anomaly-triggered PRIMARY — any reading outside the birth-world span reopens the block's assumption enumeration immediately; the monthly audit is the backstop and additionally re-derives ONE block's assumptions blind from its spec and diffs (A4, A6). Zero new cadences.
4. **Computed registry:** no standing file. At each monthly audit, grep the assumption-atom triples across all blocks → generated concentration view (shared denominators appearing in ≥2 instruments get flagged). #43b applied to the design: compute the view, don't curate it.
5. **Promotion trigger (measured, K3's own):** if material blocks >12 OR lucky-catches >1/quarter, build the standing registry file. The heavier form is earned by data, not adopted by taste.

**Success metric (A7):** catches that would otherwise have stayed lucky — channel-tagged so the sweep/luck ratio is computable. **Honest floor (K3 Q6, stated verbatim as required):** this design systematizes DERIVABLE blind spots only; the orthogonal channels (principal's input stream, cross-family referee) remain irreplaceable and are load-bearing parts of the system, not accessories.

**Fluidity metadata:** adopted 2026-07-19 (v2; K3 review ADOPT-WITH-AMENDMENTS, all A1-A8 absorbed; registry-file→computed-view divergence is the house's, reasoned from K3's own rot-ranking). Falsifier: if two quarterly cycles produce ZERO non-overlap findings from blind adversaries AND zero collision-sweep amendments, the machinery is theater — retire to the lean template-fields-only form. First re-eval: 2026-10-19.
