# HIDDEN-AI-APPLICATIONS DISCOVERY PROGRAM — demand-side sibling of the oblivious-layer program (codified 2026-07-02)

**User directive (2026-07-02):** research surfacing *existing successful* AI applications under the majority radar, NOT the chip stack. "plan and build it" — authority granted (Rule #11d cleared).

**Design conditions (from the 2026-07-02 methodology exchange — all five must hold per candidate):**
1. **Un-labeled success** — the AI deployment shows in operating numbers (margin, revenue/employee, loss ratio, cycle time), NOT in AI narrative. Screen for the effect, verify absence of the word.
2. **Wrong-sector wrapper** — covered by insurance/industrials/staffing/domestic analysts, invisible to AI analysts (Sims-SLS mechanism generalized; the one edge that survived oblivious-layer run #1's red team).
3. **Outside-disclosure evidence** — job-posting deltas, pricing-with-stable-margin, traffic/cohort data; ≥2 quarters already IN reported numbers (pilots don't count — 88% fail per `wiki/agentic-ai-enterprise.md`; screen for the 12% patterns already in the P&L).
4. **Moat ≠ model** — proprietary data / distribution / regulatory / workflow lock-in owns the surplus; standing falsifier = next frontier-model release (or API price cut) erases the advantage → customer keeps the surplus.
5. **Obliviousness verified natively, never assumed** — R1-style retail/listicle/native-language sweep as a KILL GATE (oblivious-layer run #1 lesson: Yamaichi died the moment the JP dig ran).

**Program-level falsifier:** if delta-screens surface hits only in megacaps/privates, listed under-the-radar AI-application success doesn't exist as a set — that finding feeds the END-DEMAND-DURABILITY model gap (P1) instead. Valid outcome, not failure.
