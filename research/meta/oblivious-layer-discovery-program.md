## Obliviousness scoring rubric (leg-4 codification — measured, not vibes)

Each candidate layer/name scored 0-5 per dimension (higher = more oblivious):
1. **Sell-side coverage count** (0 = >20 analysts; 5 = 0-2 analysts or no listed pure-play)
2. **Multiple-vs-segment-growth gap** (0 = multiple already prices the segment growth; 5 = multiple ignores it)
3. **News-reaction test** (0 = pure-play moves on layer news, e.g. Ibiden +19% on CoPoS; 5 = flat on load-bearing layer news)
4. **Passive/ETF membership** (0 = in SOXX/SMH/theme ETFs; 5 = in none)
5. **Information-chain stage** (0 = orders/revenue visible; 5 = physics/paper stage only)

**Investable-obliviousness threshold:** score ≥15/25 AND at least one TSE/US/EU-investable expression (or a dated path to one — IPO/spin-off). KRX reference-only, TWSE/HK flag, per the standing filter.

## Program-level falsifiers (anti-decorative, Rule #18)
1. If EVERY wave-1 nomination scores <10/25 (i.e., already discovered) → the pipeline STILL enters too late; move detection another step upstream (pre-print servers, grant filings) or retire.
2. If wave-2 red team kills 100% of candidates two runs consecutively → the "oblivious layer" framing itself is wrong for this regime (efficient discovery); log as a lesson, not a loop.
3. If a nominated layer re-rates BEFORE the program completes its scoring (measurable via the news-reaction test) → timing evidence that even this apparatus is too slow; compress wave cadence.
4. Bottleneck snap-back falsifier carried over: if the binding constraint returns upstream (wafers/power/HBM), oblivious-layer candidates underperform the existing book — program output feeds watchlist/deep-dig only; sizing stays user-gated (Rule #11).
