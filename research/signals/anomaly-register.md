# ANOMALY REGISTER — the "interesting and uncommon" ledger (born 2026-07-13, user-designed)

**What this is:** a segment-agnostic, thesis-agnostic collection of UNCOMMON observations across the whole AI theme. Entry bar = "surprising / doesn't fit an existing frame" — explicitly NOT "investable." This file exists because the harness's promotion machinery is segment-gated (Rule #6/#29) for good actionability reasons, which structurally suppresses cross-domain novelty; and Leg B's digest filter discards the uncommon-but-unclassifiable daily. User framing (verbatim-adjacent, 2026-07-13): "anything that is interesting or uncommon... a pattern that appeared independently from n sources being pattern matched."

**Mechanics:**
1. **Intake:** Leg B returns a second bucket (2-3 items/day max): "anomalies — uncommon, no thesis contact." Main-session catches and user-shared oddities also enter. One line each: date | item | source+tier | why-uncommon.
2. **WEEKLY CLUSTERING PASS (the LLM-native move):** one agent reads the full register hunting INDEPENDENT CO-OCCURRENCE — n≥3 entries from different segments/geographies/source-classes that rhyme at the mechanism level despite surface dissimilarity. Output → PC-candidate entries in `meta/cross-domain-pattern-register.md` with falsifiers. First pass: 2026-07-20, then weekly.
3. **Gates:** the SEGMENT gate stays closed for action (an anomaly-cluster cannot touch a thesis until per-segment validation per Rule #6); it is OPEN for discovery (clustering ignores segments by design). Entries age out at 90 days untouched-by-any-cluster (moved to an archive section, not deleted — late rhymes happen).
**Falsifier for the register itself (re-eval 2026-09-13):** if 8 weekly passes produce zero PC-candidates that survive their own falsifiers, the register is a curiosity shoebox — retire or fold into Leg B notes.

## Mechanics addendum (2026-07-13, user question on sources): TWO valid triangulation types feed this register
