## Method (per name — the pipeline order is load-bearing)

1. **HARD DATA FIRST (L29/Principle #1):** bottoms-up from numerical/alternative data — TSMC monthly revenue prints, contract-price QoQ (DRAM/NAND/HBM), Korea semiconductor export stats, SEAJ tool billings, SEMI wafer shipments, MLCC ASP hikes, FX quarter-averages, prior-quarter actuals of cohort peers (e.g., Micron's real print as memory ground truth), company's own sold-out/LTA statements.
2. **CONSENSUS LAST (L1/MVCRC step 8):** street numbers ingested only AFTER the bottoms-up view exists — consensus is the BAR (what's priced), never the anchor. L22: the beat that matters is vs analyst consensus, not company guide; check whether consensus sits above/below/inside the guide.
3. **LLM-NATIVE PREDICTION (mandatory components):**
   - **Revenue** (point + band)
   - **EPS** (point + band; GAAP vs non-GAAP basis stated; KR/JP names: operating profit as the convention, EPS where meaningful)
   - **Guidance** (the vintage-distribution per L13: P(raise current-FY) / P(next-FY framing) / P(hold); expected new range where applicable)
   - Plus: T+24h stock-reaction band (L14/L21/L23 stage+regime frameworks; L20 macro-confounder noted), and 2-3 falsification conditions.
4. **ENSEMBLE where sizing-consequential (Rule #17):** held names + names that gate a position decision get N=3-5 independent judges (modal + spread reported); non-held systemic reads (e.g., TSMC as cohort barometer) may be single-pass with explicit hedge.
5. **PRE-REGISTER:** one `predictions/YYYY-MM-DD-{TICKER}-{quarter}-earnings-prediction.md` per name + stub in `grading-log.md` (status: pending) BEFORE the print date.
6. **RESOLVE:** at each print, GRADE all three components (rev/EPS/guidance) + reaction; 3-layer diagnostic; lesson appended to `lessons.md` if generalizable. No un-graded predictions — the resolution IS the product (meta-posture: predictions are data-generation instruments, not performance art).
