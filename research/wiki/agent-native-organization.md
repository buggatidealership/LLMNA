## The resulting shape: HUB-AND-LEDGER, not pyramid (my model 🟡)
Thin persistent core = versioned institutional memory (the repo is the company's body); ephemeral task-swarms from a role library; verification layer proportionally larger than any human audit function; 1-few humans at the values/capital boundary. **Two alien properties:** (1) the firm is FORKABLE (identity = files → M&A = clone, competition = branch divergence); (2) firm size decouples from output (Coase: coordination-cost collapse → BOTH tails win — 1-person firms with 1,000-person output AND context-moat giants; headcount stops being the moat, context/data becomes it).
